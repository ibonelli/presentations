Que hace el frontend

  • Toma los datos del backend, y se los muestra al usuario final
  • Esto puede ser dentro del mismo framework (por ejemplo Twig Drupal rendering).
  • O desacoplado como cuando usamos:
    • En el backend REST APIs, JSON API o GraphQL.
    • Y en el frontend Angular, React, Vue.

En el modelo desacoplado hay mucho de comunicación HTTP:

  • GET POST, UPDATE, PATCH & DELETE
  • Tipos respuestas: 2xx, 3xx, 5xx
  • Todo asíncrono y llamando a una API.
  • Gatsby también lo hace pero en el building time
  • NextJS: Pre-rendering, static generation (SSG) and server-side rendering (SSR)
